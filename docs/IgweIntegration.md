**Overview**
------------

The CFO Context API returns a single, CFO-ready snapshot used for pre-call preparation and portfolio monitoring.

The response includes:

*   Interpreted financial context
    
*   Alerts and open decisions
    
*   Recent changes
    
*   A narrative headline
    
*   ROI and value tracking for demo purposes
    

All data is deterministic and demo-safe.

**API Endpoint**
----------------

**URL:**

GET /api/clients/{client\_id}/context

**Example:**

http://127.0.0.1:8000/api/clients/client\_acme/context

**Method:**

GET

**Description:**

Returns a JSON snapshot representing the current financial and operational context for a client, including value generated by system actions.

**Response JSON Structure**
---------------------------

**Top-Level Fields**

*   String. Client identifier.
    
*   ISO timestamp indicating when the snapshot was generated.
    
*   High-level narrative summary derived from alerts and metrics.
    
*   List of alert objects highlighting financial risks or issues.
    
*   Dictionary of key financial metrics such as cash balance, burn rate, runway months, revenue growth, etc.
    
*   List of notable recent changes for quick situational awareness.
    
*   List of pending decisions requiring CFO attention.
    
*   Summary of the most recent client interaction.
    
*   List of logged system actions that represent time saved and dollar value generated.
    
*   Aggregate USD value generated by the system for this snapshot.

**Value and ROI Logic (For Frontend Display)**
----------------------------------------------

Each value\_event represents a system action that replaces a manual task.

Fields:

*   **minutes\_saved** maps to estimated analyst time saved
    
*   **value\_usd** is calculated using a $45/hour baseline
    
*   **total\_value\_generated** is the sum of all value events
    

Frontend can use this to display:

*   Daily or session-based value created
    
*   ROI counters or progress indicators
    
*   Practice Leverage or Value Dashboard components
    

**Integration Notes**
---------------------

*   **CORS:** Local testing works out of the box. Headers can be added later if needed.
    
*   **Path Parameter:** Use client\_id directly in the URL. All demo client IDs return the same structured data.
    
*   **Stateless:** Each request generates a fresh snapshot.
